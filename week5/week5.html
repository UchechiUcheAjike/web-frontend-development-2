Testing and Debugging

Errors, exceptions, and warnings

When something goes wrong in a program and error occurs

System error ― there’s a problem with the system or external devices with which the program is interacting.

Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.

User error ― the user has entered data incorrectly, which the program is unable to handle.

An exception is an error that produces a return value that can then be used by the program to deal with the error

When a method is called and it does not exist, an exception error occurs

The importance of testing and debugging

It is important that errors are found, not minding Javascript is a forgiving programming language.

We do not want our code failing silently and making the errors difficult to spot and longer to track down.

Strict mode

strict mode considers coding practices that were previously accepted as just being 'poor style' as actual errors.

Using strict mode helps in clarity, and speed of programming as their are always improved ways of doing things.

Debugging in the browser

Bugs are issues in a code. Debugging is finding codes issues and making them go away.Breakpoints are a great way to debug codes

Some good debugging methods include the use of alert method, and using the console.

Error objects

An error object can be created by the host environment when an exception occurs, or it can be created in the code using a constructor function

Some error objects are: EvalError, RangeError, ReferenceError, SyntaxError, TypeError, URIError, InternalError

Throwing exceptions

Javascript can automatically throw error exceptions. Programmers can also throw their exceptions using the throw statement.

This will allow for any problems in your code to be highlighted and dealt with, rather than lurk quietly in the background

Exception handling

When an exception occurs the program most of the time terminates which may seem like everything is broken.

Exceptions can be handled better before everything crashes by catching the errors.
